# 分层规则扩展目录

> 本文件是**扩展导航**，不包含具体约束。
> 目的：为使用者提供「可以扩展哪些维度」的地图，按需在 `.cursor/rules/` 下创建对应规则文件即可。

---

## 一、分层规则体系概述

```
根 .cursorrules                ← 全局核心行为准则（语言/方案优先/决策树/Memory Bank/执行协议）
  └── project/.cursorrules    ← 项目级标准（目录约定/开发流程/交付门槛）[可选]
        └── .cursor/rules/    ← 功能模块分层规则（按数字前缀编号，按需创建）
```

**约定**：
- 规则文件以数字前缀排序（如 `20-backend-api.mdx`、`35-frontend.mdx`）
- 数字段按关注域分组，便于查找与扩展
- 每个文件聚焦单一关注域，避免跨域混写

## 三、创建新规则文件的建议结构

```markdown
# [规则名称]

> **适用范围**：[子项目 / 模块 / 全仓库]
> **关键约束**：[1-2 句核心约束，帮助快速理解]

---

## 一、[核心规范]
[具体约束]

## 二、[操作约定]
[具体约定]

## 三、检查清单（可选）
- [ ] 约束点 1
- [ ] 约束点 2

## 四、项目扩展提示（可选）
[留给使用者的扩展空间说明]
```
